Gioco is an easy to implement gamification gem based on the concept of "plug and play".
It doesn't matter if you already have a fully functional database, Gioco will
smoothly integrate
everything and provide all methods that you might need.
Documentation

Concept
Gioco implements the concepts of Badge, Points and Types to cover the majority of gamifications implementations. Points and Types are optional but can be very useful.
The basic logic is that a Resource ( e.g. User ) has Points of different Types and when this Resource gets a certain amount of Points of a specific Type it can win or lose Levels, connecting or disconnecting the Resource with a Badge ( getting a badge through a level ).
But Points are optional so, if you don't want to register Points, all you have to do is to add and remove Badges as required by your business rules ( through a level ).

Install and Setup
Gioco is available through Rubygems, to install it, just add Gioco in your gemfile:
gem 'gioco'
and run Bundler:
> bundle install

After installing Gioco you'll need to set it up. The MODEL_NAME should be replaced for the model you want the resource of the gamification to be. (e.g. user )
Gioco has two optional setup parameters, --points and --types, they can be used togheter or not.
rails g gioco:setup --points --types
other example would be:
rails g gioco:setup
The optional --points argument will setup Gioco with a points system explained at the Concept. And the optional argument --types provides an environment of multiple types of badges and points ( If the option --points is being used too ).

Badges Management
The badges are managed through Rake tasks, enabling you to add and remove badges as you need
The boolean DEFAULT option is responsible for adding a specific badge to all your current resourcea regsstrations.
To add Badges, use:
With --points option:
rake gioco:add_badge[BADGE_NAME,POINTS_NUMBER,DEFAULT]
With --types option:
rake gioco:add_badge[BADGE_NAME,TYPE_NAME,DEFAULT]
With --points and --types option:
rake gioco:add_badge[BADGE_NAME,POINTS_NUMBER,TYPE_NAME,DEFAULT]
Without --points and --types option:
rake gioco:add_badge[BADGE_NAME,DEFAULT]
And to remove Badges, use:
With --types option:
rake gioco:remove_badge[BADGE_NAME,TYPE_NAME]
Without --types option:
rake gioco:remove_badge[BADGE_NAME]


Methods
Gioco provides some methods that will allow you to easily implement your gamification logic inside your application.

Resource
Change Points This method is useful only when you setup the Gioco with the points system.Type_id should be used only when you already used it as a setup argument

user = User.find(1)
user.change_points({ points: Points, type_id: Type_id })
If you have setup Giogo without --type then you shoul only pass the points argument instead of a hash:

user = User.find(1)
user.change_points(Points)
Next Badge? Return the next badge information, including percent and points info. Type_id should be used only when you already used it as a setup argument

user = User.find(1)
user.next_badge?(Type_id)
If you have setup Giogo without --type then you shouldn't pass any argument:

user = User.find(1)
user.next_badge?()
Get Badges and Points To get the badges or points of you resource all you have to do is:

user = User.find(1)
user.badges
user.points

Badges
Add Add a Badge to a specific resource, it will return the badge added, or if you are using points system it will return a hash with all badges that ahd been added.

badge = Badge.find(1)
badge.add(Resource_id)
Remove Remove a Badge of a specific resource, it will return the badge removed, or if you are using points system it will return a hash with all badges that ahd been removed

badge = Badge.find(1)
badge.remove(Resource_id)

Ranking
Generate
Gioco::Ranking.generate
This method returns an array with a ranking of all your resources ordered by points or quantity of badges, the result format will change according to the setup arguments you have used ( --points and/or --types )
