Gioco is an easy to implement gamification gem based on the concept of "plug and play".
It doesn't matter if you already have a fully functional database, Gioco will
smoothly integrate
everything and provide all methods that you might need.
Documentation

Concept
Gioco implements the concepts of Badge, Level, Points and Types to cover the majority of gamifications implementations. Points and Types are optional but can be very useful.
The basic logic is that a Resource ( e.g. User ) has Points of different Types and when this Resource gets a certain amount of Points of a specific Type it can win or lose Levels, connecting or disconnecting the Resource with a Badge ( getting a badge through a level ).
But Points are optional so, if you don't want to register Points, all you have to do is to add and remove Badges as required by your business rules ( through a level ).

Install and Setup
Gioco is available through Rubygems, to install it add Gioco in your gemfile:
gem 'gioco'
and run Bundler:
> bundle install

After installing Gioco you'll need to set it up. The MODEL_NAME should be replaced for the model you want the resource of the gamification to be. (e.g. user )
Gioco has two optional setup parameters, --points and --types, they can be used togheter or not.
rails g gioco:setup MODEL_NAME --points --types;
or
rails g gioco:setup MODEL_NAME;
The optional --points argument will setup Gioco with a points system explained at the Concept. And the optional argument --types provides an environment of multiple types of badges and points ( If the option --points is being used too ).

Badges Management
The badges are managed through Rake tasks, enabling you to add and remove badges as you need
The boolean DEFAULT option is responsible for adding a specific badge to all your current resource registrations.
To add Badges, use:
With --points option:
rake gioco:add_badge[BADGE_NAME,POINTS,DEFAULT]
With --types option:
rake gioco:add_badge[BADGE_NAME,TYPE,DEFAULT]
With --points and --types option:
rake gioco:add_badge[BADGE_NAME,POINTS,TYPE,DEFAULT]
Without --points and --types option:
rake gioco:add_badge[BADGE_NAME,DEFAULT]
And to remove Badges, use:
rake gioco:remove_badge[BADGE_NAME]


Methods
Gioco provides some methods that will allow you to easily implement your gamification logic inside your application.

Resource
Change Points
Gioco::Resources.change_points( Resource_id, Points, Type_id )
This method is useful only when you setup the Gioco with the points system.Type_id should be used only when you already used it as a setup argument

Badges
Add
Gioco::Badge.add( Resource_id, Badge_id )
The Badge.add method is responsible for adding a specific badge to a resource.

Remove
Gioco::Badge.remove( Resource_id, Badge_id )
The Badge.remove method is used to remove a badge of a resource.

Ranking
Generate
Gioco::Ranking.generate
This method returns an array with a ranking of all your resources ordered by points or quantity of badges, the result format will change according to the setup arguments you have used ( --points and/or --types ):br />
Application
Inside your application you might have to get Badges, Levels or Points data. And you are able to do this using (replace RESOURCE_NAME for your resource model name) :
RESOURCE_NAME.badges
RESOURCE_NAME.levels
RESOURCE_NAME.points
